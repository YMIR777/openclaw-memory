# Memory Index

## Purpose

This directory is split by function so daily notes, long-lived topic files, and archived material do not drift back into one flat pile.

## Active Layers

- `YYYY-MM-DD.md`: active daily notes
- `topics/`: durable subject files
- `projects/`: project-specific long-lived context
- `hot/` and `warm/`: staged memory layers already in use
- `.dreams/` and `dreaming/`: memory-system internals; do not reorganize casually

## Archive Layers

- `archive/daily-2026-q2/`: archived daily logs from April and May 2026
- `dreams-archive/`: retired dream diary material

## Rules

1. Keep only recent daily notes at the top level.
2. Promote stable knowledge into `topics/` or `projects/`.
3. Archive old daily notes by dated bucket, not as loose files.
4. Treat `.dreams/` and `dreaming/` as operational internals unless there is a specific memory-system task.
