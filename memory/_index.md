# Memory Index

## Purpose

Memory is split by function so daily notes, long-lived subject files, and archived material do not drift back into one flat pile.

## Active Layers

- `YYYY-MM-DD.md`: active daily notes (keep only the most recent 2-4 weeks at root)
- `人物/`: person/relationship/self-knowledge files
- `项目/`: project-specific long-lived context, one directory per project
- `技术系统/`: tools, platforms, languages, and OpenClaw internals
- `踩坑修复/`: resolved incidents and post-mortems
- `方法论/`: workflows, principles, and development strategy
- `warm/`: staged warm-memory layer (keep only if actively maintained)
- `dreaming/`, `.dreams/`: memory-system internals; do not reorganize casually

## Archive Layers

- `归档/旧日记-2026-q1/`, `归档/旧日记-2026-q2/`: archived daily logs
- `归档/梦境/`: dreaming diary split by month
- `归档/退休热记忆/`: old hot-memory snapshots
- `归档/退休主题/`: outdated or superseded topic files

## Rules

1. Keep only recent daily notes at the top level; archive by dated bucket.
2. Promote stable knowledge into `人物/`, `项目/`, `技术系统/`, `踩坑修复/`, or `方法论/`.
3. Keep the bilingual `CATEGORY-主题.md` naming style: English category prefix for sorting and search, Chinese subject for readability.
4. Treat `.dreams/`, `dreaming/`, and `dreams-archive/` as operational internals unless there is a specific memory-system task.
