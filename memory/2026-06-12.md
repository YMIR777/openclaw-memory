## 💡 核心教训：Claude Code Skills 加载机制

**之前的错误假设：** Claude Code 启动时加载所有 SKILL.md → 21 skills = 70K token

**真相（两个独立来源确认）：**
- Skills 使用**渐进式披露（progressive disclosure）**
- 启动时只加载 skill 名称 + 一句话描述（~几KB）
- 识别到触发条件时才加载完整 SKILL.md
- 不用的 skill 零上下文占用

**结论：装 100 个 skill 和装 1 个的启动成本几乎一样。以后看到有趣的 skill 直接装，不用纠结。**
