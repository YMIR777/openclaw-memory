## 📚 Mac 自动化控制学习笔记

### 当前方案的问题（VNC 方案）
1. **坐标不准确** - 截图缩放、Retina 屏幕导致坐标偏差
2. **每次操作都重新连接** - 已用守护进程解决部分问题
3. **不稳定** - 守护进程容易断开

### 发现的新方案

#### 1. AppleScript + Python (推荐)
- **wemac** 项目：https://github.com/x5iu/wemac
- 使用 AppleScript 控制微信 UI
- 使用 PyUserInput 模拟鼠标点击
- 更精确的 UI 元素定位

#### 2. Accessibility API (pyax / atomacos)
- **pyax**: https://github.com/eeejay/pyax
- **atomacos**: https://github.com/daveenguyen/atomacos
- 直接访问 macOS Accessibility API
- 可以精确定位 UI 元素（按钮、输入框等）
- 不依赖坐标

#### 3. pyUIauto
- https://github.com/harveyf2801/pyUIauto
- 跨平台 UI 自动化库
- 结合了 pywinauto (Windows) 和 atomacos (Mac)

#### 4. macOS-use 项目
- https://github.com/browser-use/macOS-use
- 最新最火的 AI 控制 Mac 项目
- 需要配置 LLM API

### 改进方向

1. **短期**：使用 AppleScript 做微信自动化
   - 安装 PyUserInput: `pip3 install PyUserInput`
   - 编写 AppleScript 脚本控制微信

2. **长期**：使用 Accessibility API
   - 学习 pyax 或 atomacos

### 重要发现：坐标系统

**Retina 屏幕的双重分辨率：**
| 工具 | 分辨率 | 比例 |
|------|--------|------|
| VNC (屏幕共享) | 2880 x 1800 | 2x |
| PyUserInput / pyautogui | 1440 x 900 | 1x (实际) |

**坐标转换：**
- VNC 坐标 → pyautogui: 除以 2
- pyautogui → VNC: 乘以 2

### 中文输入解决方案

**问题：** PyUserInput / pykeyboard 不支持中文输入

**解决方案：**
```python
import pyperclip
import pyautogui

# 1. 复制到剪贴板
pyperclip.copy("你好")

# 2. 粘贴
pyautogui.hotkey('command', 'v')
```
