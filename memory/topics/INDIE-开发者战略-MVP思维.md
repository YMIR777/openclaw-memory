## 🎯 核心理念：快速交付，不要完美

### MVP 优先
- 做一个**最小能用的版本**，只有最核心的功能
- 先上线用，根据真实反馈迭代，不脑补用户需求
- 大金库 MVP = 收入录入 + 支出录入 + 当月总览 + 历史列表
- **不要在第一版做的事：** 粒子特效、多账户、预算规划、趋势图

### 快速交付
- 给自己设死线（deadline），做不到就砍功能，不延长工期
- vibe coding 优势是速度，不要陷入"再改改完美一点"
- 参考节奏：Day1 生成框架 + 核心页面，Day2 调通数据流，Day3 测试 + 上线

### 失败要快，成功要更大
- 一个产品 2-4 周跑不起来或没人用 → 放弃，继续下一个
- 不要在错误项目上消耗
- 每个项目都是练习场，做完就学到东西了

---

## 🛠️ 开发前提醒检查清单

**每次开发新产品前，我要主动问小黒：**

1. "这个产品解决什么问题？"（一句话能说清楚吗？）
2. "MVP 最少需要哪几个功能？先不做哪些？"
3. "预计多久做完 MVP？超时了怎么办？"
4. "做完后怎么验证它有没有人用？"

**开始做之后提醒：**
- "不要在细节上磨太久，先跑起来"
- "这个功能是 MVP 必须的吗？不是就先跳过"
- "做完比完美重要"

---

## 📦 项目延伸思路

- 一个项目能不能长出更多东西？
- 现在单用户 → 以后能不能开放给同类用户？
- 有了数据 → 能不能做衍生内容/工具？

---

## 📣 Build in Public

- 在小红书/B站/即刻发做项目的过程
- 边做边曝光，获取早期反馈 + 建立个人品牌
