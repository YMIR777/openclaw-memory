### 最终正确架构
```
第 1 层 · anchorRef（外层 div）
  - 接收所有鼠标事件（onMouseEnter/Move/Leave）
  - 提供 getBoundingClientRect() 稳定坐标系
  - 不参与任何旋转

第 2 层 · tiltRef（中层 div）
  - DOM 直接操控 style.transform（绕开 React/Framer Motion）
  - transform 字符串内包含 perspective(800px)
  - willChange: 'transform'

第 3 层 · motion.button（内层）
  - Framer Motion 只管 scale/opacity
  - 卡片内容 + 光斑 overlay + 长按预览
```

### 标准 tilt card 公式（已验证）
```js
const fx = (mouseX - centerX) / halfWidth   // -1(left) ~ +1(right)
const fy = (mouseY - centerY) / halfHeight  // -1(top) ~ +1(bottom)
el.style.transform = `perspective(800px) rotateX(${fy * max}deg) rotateY(${-fx * max}deg)`
```

### 踩过的 6 个坑
1. perspective 放 CSS 属性 → 必须放 transform 串里
2. rotate3d() → 用标准 rotateX/rotateY
3. 鼠标事件挂旋转层 → 挂外层稳定 div
4. React state 驱动 tilt → DOM 直接操控
5. tilt 层放内容副本 → tilt 层只放视觉 overlay (pointerEvents:none)
6. 反复推导公式 → **应先搜索已验证实现**
