## 🚨 QMD 严重故障（03:03 发现）

**问题**：QMD memory search 完全失效！

**症状**：
```
[memory] qmd collection add failed: Module not found 
"/Users/Ymir/.bun/install/global/node_modules/@tobilu/qmd/dist/cli/qmd.js"

Memory Search: Provider: qmd (requested: qmd)
Indexed: 0/14 files · 0 chunks
```

**根因**：qmd 从源码安装（bun install），但没有 build。缺少 `dist/cli/qmd.js` 文件。
- qmd 的 `bin/qmd` 脚本根据 bun.lock 判断用 bun 执行 `dist/cli/qmd.js`
- 但 `dist/` 目录不存在（从未 build）

**验证**：
```bash
# 检查 dist 目录
ls ~/.bun/install/global/node_modules/@tobilu/qmd/dist/  # 不存在

# 检查 qmd 能否运行
cd ~/.bun/install/global/node_modules/@tobilu/qmd && bun bin/qmd --help
# → Syntax Error

# 测试 memory search
openclaw memory search "test"
# → No matches (返回空)
```

**可能的修复方案**：
1. 在 qmd 目录运行 `bun build` 编译 dist/
2. 或者重新用 npm 安装 qmd（npm 会 build）
3. 或者检查是否可以用 tsx 直接运行 src/cli/qmd.ts

**影响**：
- 语义搜索功能完全不可用（0 chunks indexed）
- 但 keyword search 可能仍可用（待验证）
