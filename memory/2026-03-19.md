## 深刻反思：为什么我无法真正驱动 Claude Code

### 问题
用户明确要求用 Claude Code + SpecKit 开发项目，但我无法真正做到。

### 根本原因

**1. 我创建的 Claude Code skill 是"参考手册"而非"集成工具"**
- SKILL.md 里的内容告诉我怎么用 Claude Code
- 但 OpenClaw 并没有真正调用 Claude Code 的能力
- 这就像买了一本菜谱，但厨房里没有灶台

**2. `sessions_spawn` 的 `runtime: "acp"` 不等于 Claude Code**
- 错误信息：`ACP runtime backend is not configured`
- 即使装了 `openclaw-acp` 插件，它其实是"Agent Commerce Protocol"插件
- 不是用来驱动 Claude Code 的

**3. `sessions_spawn` 的 `runtime: "subagent"` 用的是我自己的模型**
- 不是 Claude Code，是 MiniMax-M2.7
- 所以项目是用我的模型建的，不是 Claude Code

**4. Claude Code CLI 本质上是交互式工具**
- slash 命令（`/speckit.*`）需要实时终端交互
- `claude -p` 打印模式不支持 slash 命令
- 我无法通过管道或后台进程真正控制它

### 正确做法

**方案 A：ACP Runtime（已验证可用）**
1. 安装 acpx-orchestrator：`clawdhub install acpx-orchestrator`
2. 编辑 `~/.openclaw/openclaw.json`，在 `plugins.entries` 添加 `"acpx": { "enabled": true }`
3. 重启 gateway：`openclaw gateway restart`
4. 调用方式：`sessions_spawn` 使用 `runtime: "acp"` + `agentId: "claude"`

**方案 B：用户手动终端操作**
1. 打开终端
2. `cd /Users/Ymir/Documents/vibe-projects/spec-kit-setup`
3. `claude` 启动 Claude Code
4. 输入 `/speckit.*` 命令

**关于 CLAWHUB_TOKEN 配置：**

### ClawHub Rate Limit 解决方案
**问题：** `clawdhub install` 一直报 "Rate limit exceeded"
**解决：** 用 token 方式登录而不是浏览器授权
```bash
clawhub login --token <your-token>
```
注意：不要在命令中明文输入 token，可能有安全风险。建议用环境变量或配置文件方式。

## StarUML 自动化方案

Claude Code 研究了 StarUML 自动化，发现两种方案：

### 1. StarUML CLI（仅 v5.x+）
```bash
$ alias staruml='/Applications/StarUML.app/Contents/MacOS/StarUML'
$ staruml --help
```

### 2. staruml-controller 扩展（推荐）
- 仓库：https://github.com/pontasan/staruml-controller
- 安装：克隆到 `~/Library/Application Support/StarUML/extensions/user/staruml-controller`
- 启动：Tools > StarUML Controller > Start Server...（默认端口 12345）

**API 示例（用例图）：**
- `POST /api/usecase/diagrams` - 创建用例图
- `POST /api/usecase/actors` - 创建 Actor
- `POST /api/usecase/use-cases` - 创建 Use Case
- `POST /api/usecase/associations` - 创建关联线
- `PUT /api/views/:id` - 调整位置（left/top/width/height）

**用途**：程序化生成 UML 图、自动化用例图设计等

## Claude Code + ACP Runtime 配置总结（2026-03-19）

### 核心配置
1. 安装 acpx-orchestrator：`clawdhub install acpx-orchestrator`
2. 启用插件：编辑 `~/.openclaw/openclaw.json`
```json
"plugins": {
  "entries": {
    "acpx": { "enabled": true }
  }
}
```
3. 重启 gateway：`openclaw gateway restart`

### 正确调用方式
```javascript
sessions_spawn(
  task="task description in English",
  agentId="claude",
  runtime="acp",
  cwd="/path/to/project",
  mode="run",
  timeoutSeconds=600
)
```
- ❌ `runtime: "subagent"` = 用 OpenClaw 自己的模型
- ✅ `runtime: "acp"` + `agentId: "claude"` = 真正调用 Claude Code

### ClawHub Rate Limit 解决方案
```bash
clawhub login --token <token>
# 或设置环境变量
export CLAWHUB_TOKEN=clh_xxx
```
不要明文在命令里输入 token。
