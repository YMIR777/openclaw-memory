## 💡 核心经验教训

### 1. Canvas 画图工具开发流程规范

**正确流程应该是：**
1. 明确图形类型（圆形/矩形/线条）
2. 每种图形单独测试，确认渲染正确
3. 确认拖拽时：起点、终点、中心点、半径/宽高的计算正确
4. 合入主逻辑

**这次的问题：** 先做功能再调样式，功能和样式混在一起调试，容易顾此失彼。

### 2. 视觉类项目如何验收

不能只靠"能打开控制台看 console"——必须实际看到渲染结果。
- 截图验证是最直接的方式
- 如果是 headless 环境无法截图，至少要能描述清楚"预期看到什么"

### 3. 提交前自检清单

- [ ] 功能逻辑：点击生成图形 ✓，拖拽生成图形？
- [ ] 视觉效果：颜色、位置、大小是否符合预期
- [ ] 边界情况：图形超出画布怎么办、点击和拖拽冲突吗
- [ ] 最终输出：截图确认实际渲染效果
