# 绮梦账间 — 复盘总结

> 日期：2026-04-07

---

## 一、失败根因

### 1. 过度并行导致全面崩溃

我同时启动了 8 个 frontend agents，全部共享同一个 MiniMax API Token。结果：
- 第 1 个 agent 开始消耗 token 配额
- 其他 7 个 agent 全部遇到 HTTP 429 Rate Limit
- 所有 agent 变成 error 状态，任务全部停在半成品

**正确的做法**：最多 2 个 agent 并行，串行开发核心功能，确保每个功能完整交付后再开发下一个。

### 2. 需求总量 vs 团队容量

绮梦账间的设计非常复杂，包含了：
- 双模式粒子引擎（深夜/暖白）
- 决策点系统（支出审判交互）
- 灵种（时薪生命体）
- 目标星体可视化
- 魔法输入框 + 语义识别
- 陪玩时间轴星辰布局
- 粒子实验室参数面板
- 声音炼金术
- Loading + Intro + Dashboard + Income + Stats 5个完整页面

**这些是 2-3 个人月的设计工作量**，我一个 session 里全部丢给 agents 做，每 agent 只分配了几分钟。结果：每个功能都做了一点，但没有一项是完整的、精良的。

### 3. 视觉实现 vs 视觉设计是两回事

我犯了一个错误：把"设计思路"当成"实现能力"。绮梦账间的设计思路（点云粒子、决策点、灵种星云）确实很有创意，但把这些设计变成真正精致的视觉效果，需要：

- 每个动画曲线精心调参（缓动、时长、阻尼）
- 粒子系统的颜色/密度/运动轨迹的美学把控
- UI 元素的间距/字重/留白的视觉平衡
- 多次迭代和用户测试

agents 在几分钟内生成的代码，只能做出"能跑的功能 Demo"，不是"精致的产品"。

### 4. 架构一致性缺失

8 个 agents 同时写代码，没有统一的代码规范和架构决策：
- 有些用 `use client`（Next.js 语法），有些不用
- 组件拆分逻辑不统一
- 状态管理分散在多个 store 文件
- 没有统一的动画风格和设计 Token

---

## 二、需求是否过多？

**不是。** 需求本身没问题，问题在于我在错误的时间用错误的方式做了这些需求。

绮梦账间是一个**作品集级别**的项目，它的复杂度是合理的。但正因如此，它需要：
- 完整的开发周期（不是一次对话里全部搞定）
- 精心打磨的每一帧动画
- 多次视觉迭代

我应该这样做才对：
1. **先完成一个最小的可用版本**（Loading + Intro + Dashboard 基础框架）
2. **逐个功能添加**，每加一个功能都确保质量
3. **而不是一次性丢出 8 个 agent 全部开发**

## 三、经验教训

### 技术层面
1. **API Rate Limit 是真实约束**：MiniMax 有并发限制，一次对话内不要超过 2 个并行 agent
2. **Vite 不支持 `'use client'`**：这是 Next.js 专有指令，下次要注意框架兼容性
3. **粒子引擎需要专项调试**：R3F 的粒子系统实现细节（bufferAttribute args vs array/count/itemSize）容易出错，需要单独验证

### 项目管理层面
1. **并行不是银弹**：超过 2 个 agent 并行时，API 限制会让效率归零
2. **先完成，再完美**：每个功能完整交付一项，再做下一项
3. **设计复杂度需要工程时间支撑**：绮梦账间的视觉设计是 2-3 个月的工作，不是一个周末

### 关于 ClawTeam
1. **ClawTeam 适合独立模块开发**：每个 agent 负责完全不重叠的模块
2. **不适合同一个项目的强耦合部分**：Dashboard 的各个组件互相依赖，agent 之间没有协调机制
3. **Rate Limit 会让并行策略完全失效**：下次用 ClawTeam 先确认 API 容量

---

## 四、给之后的建议

如果你未来要重新开发绮梦账间：

1. **不要用 ClawTeam 一次性开发全部**：最多 1 个 architect + 1 个 frontend agent
2. **分阶段交付**：
   - Phase 1：Loading + Intro（纯视觉，2-3 天）
   - Phase 2：Dashboard 基础框架（粒子背景 + 主题切换，2-3 天）
   - Phase 3：魔法输入框 + 决策点（核心交互，3-5 天）
   - Phase 4：陪玩工作台（3 天）
   - Phase 5：粒子实验室（2 天）
3. **每个阶段都需要视觉 review**：做完截图给我看，不满意就继续调
4. **或者直接用 Claude Code 你自己掌控**：你自己 vibe coding 一个页面，调到自己满意，再做下一个
