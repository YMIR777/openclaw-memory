# Life OS — 小黒的人生操作系统

**创建日期：** 2026-06-26  
**定位：** 存放宏大叙事、人生地图、时间线取证、四象限证据库与长期自我理解材料。  
**与 Dr. Sharp 的关系：** Life OS 存地图，Dr. Sharp 动手术。

---

## 核心原则

> **允许宏大叙事点火，但必须在 24 小时内落成一个证据。**

宏大叙事不是敌人。它能提供方向、意义、燃料、神话感和目标指南针。  
但如果它没有落到现实证据，就会变成新的精神高潮机器。

因此：

- Life OS 可以容纳完整地图；
- Dr. Sharp 只调用与当前问题有关的证据；
- 每次宏大模型输入后，必须产生一个可保存、可验证、可复盘的小证据。

---

## 分层设计

### 1. Life OS：地图层

存放：

- HUMAN 3.0 / 四象限 / 发展模型；
- 人生采访与时间线取证；
- 童年、家庭、学校、关系、钱、身体、创作等原始材料；
- 宏大叙事燃料库；
- 个人知识库与长期自我理解。

Life OS 的目标不是立刻治疗，而是建立长期可调用的“人生地图”。

### 2. Dr. Sharp：手术层

存放：

- 当前核心循环；
- 强判断与归因校验；
- 四象限约束诊断；
- 14 天证据链；
- 每次分析后的现实动作。

Dr. Sharp 不背整个地图，只在需要时从 Life OS 调证据。

### 3. Memory Topics：稳定结论层

当某个判断经过多次验证，才沉淀进长期记忆。

例如：

- “Vocation 小建造能缓解关系焦虑”；
- “童年某模式解释了父亲凝视”；
- “某个宏大叙事对小黒有效/无效”。

## 使用协议

### 当输入宏大模型时

1. 先完整理解；
2. 判断哪些部分适合小黒；
3. 降噪：删除身份刺激、玄学化、过度宏大叙事；
4. 提炼成 Life OS 地图；
5. 转化成 Dr. Sharp 可用诊断工具；
6. 24 小时内落一个证据。

### 当进行人生采访时

1. 不急着从出生开始；
2. 优先关键转折点；
3. 童年作为证据层，而不是默认入口；
4. 每个事件记录：事件 / 解释 / 规则 / 循环 / 象限影响 / 主约束 / 实验 / 证据。

### 当进行 Dr. Sharp 分析时

1. 当前问题优先；
2. 强判断必须标注为工作假设；
3. 必须做归因校验；
4. 不能只给理解，必须给现实动作；
5. 若需要童年或宏大模型，再从 Life OS 调取。
