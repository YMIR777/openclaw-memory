# HEARTBEAT.md

## 心跳配置
- 检查间隔：30 分钟
- 每次检查只做最小必要操作
- 深夜（23:00-08:00）保持安静

## 要检查的项目

### 每次心跳（轻量）
- 检查是否有紧急未读消息
- 如果有重要会话，回复用户

### 知识沉淀最佳实践（按事件，不按时间）

**不要追求定时跑复杂蒸馏。** 真正的触发：

| 触发条件 | 应该做的事 |
|----------|-----------|
| 用户说"记住这个" | 立即提炼写入 Obsidian 或 memory-db |
| 完成项目/任务 | 写项目总结到 Obsidian |
| 修复重要 bug | 写 case 笔记 |
| 用户问"之前怎么处理的" | 调出相关记忆，回答并确认是否更新 |
| 开始新任务涉及旧领域 | 先搜索历史，再决定是否更新 |

**auto-capture 的定位：**
只是 raw log 捕获，不是蒸馏。
CRON 只追加日记，不做深度处理。

## 不做的事
- 不主动打扰用户
- 不跑重操作（特别是记忆蒸馏）
- 不发散探索
